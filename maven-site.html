Site publishing on gh-pages

  • make sure you have a gh-pages branch on github ready (cf https://help.github.com/articles/creating-project-pages-manually)
  • to be able to upload your project documetations to gh-pages, make sure you set your github token correction in a `github.global.oauth2Token` property.
    • cf https://github.com/github/maven-plugins#core-configuration
  • make sure you have a correct site distribution set in you pom

    You can upload your maven-site to the gh-pages on github with the following command line

    $ mvn site site:stage scm-publish:publish-scm \
  -Dscmpublish.checkinComment="the comment of your commit, change it please!"

    This will publish the staged documentation site to the gh-pages of your repository. In minutes, you will have a live website on:

  • http://zenika.github.com/zenika-parent
    • If the repository owner is Zenika,
    • If the repository name is zenika-parent

    You can automatically upload your maven-site to gh-pages branch by setting the property `inherit.parent.scm-publish` to `true`

Javadoc

By setting the property `inherit.parent.reporting` to `true`, the javadoc of your project is generated with your site. It will also be published on gh-pages on deployment so, it's maybe time to do some documentation.
